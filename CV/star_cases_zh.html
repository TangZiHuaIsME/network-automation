二、STAR 案例一：全栈网络与软件定义网络（SDN）平台专家履历

Situation（情境）

负责全中国区大型企业客户（如机场、大型医院、全国连锁门店等）的网络及数智化网管平台（eSight、iMaster NCE-Campus、CampusInsight）的运维与故障交付。在3年半的周期内，身处高强度、高对抗性的技术支持一线，作为核心技术接口人处理全国各类疑难杂症与定制化数据需求。

Task（任务）
  • 高并发交付：保持每日极高的工单消化效率，全方位保障客户大规模园区网的高可用性与业务连续性。
  • 全栈技术落地：主导网管平台系统级部署，完成从底层硬件（RAID/Bond）、平面规划，到上层SDN高级准入认证（802.1X/Portal/MAC/HWTACACS）及微服务/数据库全链路的部署、规划与调优。
  • 数据治理与自动化：突破平台前端图形化页面的功能导出限制，利用技术手段为客户高效、精准地提取现网底层核心数据。
  • 跨领域协同：面对超越单一产品边界的超大型、跨系统未知现网故障，发挥核心纽带作用，打破技术壁垒推进问题闭环。
Action（行动）
  • 高并发交付与北向 API 数据治理：在3年半内坚持每日高效处理10-15个复杂技术问题单，累计闭环超万级工单。除日常故障恢复外，针对客户大考及行为审计需求，深度利用 iMaster NCE-Campus 与 eSight 的北向 RESTful API 架构，通过 `curl`、Postman 及 Python 脚本进行 API 调用与批量轮询。成功绕过前端限制，直接从平台后端提取全网 AP 关联用户数、历史拓扑链路流量、无线准入失败审计等海量原生 JSON/XML 数据，并进行结构化清洗，转化为可视化报表交付客户管理层。
  • 全栈系统级部署与深度调优：协助客户完成网管平台的系统级规划，从最开始的画 RAID 卡、挂载磁盘、安装操作系统，再到网络南北向平面与管理平面的高可用规划及 Bond 绑定。当系统前端“爆红”或底层微服务 CPU 过高时，直接深入 Linux 系统底层通过脚本进行资源调优，并进入数据库进行表结构与数据流级别的操作，解决微服务底层逻辑故障。
  • 高级准入与跨平台对接：在 SDN 平台上基于 NETCONF、SNMP 协议落地大规模设备纳管。主导部署复杂的 Fabric/VxLAN 大二层架构及业务随行策略，攻克每日高频出现的 802.1X、MAC、Portal、HWTACACS 认证难题，成功打通网络平台与飞书、微信、第三方 SSO 单点登录系统及短信邮件网关的跨平台对接。
  • 跨领域横向拉通（核心沟通力）：面对极端复杂的跨产品疑难杂症，主动打破技术孤岛。作为核心牵头人，成功拉通包含服务器、交换机、路由器、WLAN 等多个领域的资深研发工程师、系统架构师在内的数十人专家团队进行联合会签。在多方联合会诊中，精准统一各方技术语言、高效梳理多方排查边界、主导分配排查任务，作为“技术总指挥”引导几十人的团队聚焦核心矛盾，快速收敛故障。
Result（结果）
1. 产出与绩效：3年半内以高标准高质量处理万级问题单，成功推进并解决了数起涉及多方产品的行业级重大顽固故障，将超大故障的平均闭环效率提升了30%以上，期间未发生一起因沟通不畅导致的客户投诉。
2. 数据赋能成果：通过北向API数据提取技术，帮助多个行业头部客户建立了网络行为审计与扩容基线报表，提升了客户对平台数据的二次利用价值，交付质量获得各领域专家及客户的高度赞誉。

三、STAR 案例二：大型园区大面积 Wi-Fi 网络瘫痪应急恢复与故障定位

Situation（情境）

每周均会受理全中国区大企业客户（如机场值机、医院挂号、大型连锁门店等）提出的最高优先级突发工单：整栋大楼或整个特定区域的 Wi-Fi 连接成功后无法访问网络（Wi-Fi 爆满但无网络连接）。这类故障往往影响上万人，业务敏感度极高，伴随着客户侧巨大的舆论与经营压力。

Task（任务）

作为全中国区业务的一线技术负责人，必须在极短时间内控制故障扩散、恢复业务，并在跨厂商、跨组件的复杂大环境下快速找出根因，同时提供精准的数据提取作为故障溯源依据。

Action（行动）
  • 快速响应与风险控制（应急恢复）：第一时间对接客户，快速盘点爆炸半径（影响范围）、调取近期变更记录。遵循“业务优先”原则，若处于高峰期且情况紧急，果断通过控制台或 SDN 平台将无线/有线认证策略临时调整为 Open（不认证）状态，瞬间实现业务盲区绕行，5分钟内控制故障扩散，最大化保障客户业务连续性。
  • 业务基线审计与用户流轨迹追踪（Trace）：在业务恢复或非紧急状态下，凭借丰富的现网经验，横向快速审计设备的 Portal 模板、AAA 模板、RADIUS 服务器模板等核心配置基线。随后在 SDN 平台及网络设备侧启动用户痕迹追踪（Trace 机制），精准捕捉目标测试用户的全链路认证行为轨迹。
  • 底层全量抓包根因定位：深入 SDN 平台及网网设备的 Linux 底层，使用 `tcpdump` 工具进行全量报文抓包。纵向剖析 RADIUS 请求（Request）、挑战（Challenge）、回应（Response）等交互全过程，逐个环节排查认证报文是在哪个节点丢失或哪项凭据校验失败，最终精准锁定大部分故障根因为“客户中间链路的第三方厂商防火墙安全策略拦截了认证报文”或“客户私自变更了非对等配置”。
  • 北向 API 数据溯源复盘：在故障定位及复盘阶段，为了向客户管理层提供更直观的故障影响面报告，利用 Postman/curl 工具直接调用 Campus/eSight 的北向 API 接口，批量拉取故障时间段内的“全网终端准入失败日志”与“异常 AP 状态数据”，通过对底层 JSON 数据的提取与横向对比，精准锁定受影响的用户规模与终端分布，用客观数据说话。
  • 多领域专家拉通与攻坚（核心沟通力）：若故障涉及底层代码缺陷、微服务偶发性 Bug 或跨厂商硬件冲突等极端复杂场景，单兵作战难以攻克时，立刻启动重大故障响应机制。凭借出色的技术沟通力与组织能力，迅速拉通包含 WLAN 研发、数通架构师、服务器专家及第三方厂商在内的数十人专家团队进行线上联合会诊。在顶着巨大现网压力的几十人电话会议中，高效梳理时间线，分配排查任务，防止多方推诿，确保信息在客户、研发、架构师之间透明高效流动，形成攻坚合力。
Result（结果）
1. 极致恢复效率：通过独创的“优先 Open 恢复 + 底层 Trace/tcpdump 精准定位 + 北向 API 数据溯源”组合拳，将此类高频大面积网络故障的平均恢复时间（MTTR）控制在 15 分钟以内。
2. 机制建设：通过多次组织几十名顶级专家的联合会诊，不仅彻底解决了跨厂商、跨平台的多起隐蔽重大故障，更为客户建立了一套完善的跨领域故障联合排查 SOP 机制，技术与沟通协作能力获得了各领域专家及客户的高度赞誉。

四、未来求职与高薪岗位推荐方向

您当前的技术栈属于非常抢手的“网络+平台+自动化+Linux底层”的复合型人才，建议重点瞄准以下高薪岗位：

  1. 网络自动化运维工程师 / 网络 programmability 专家：您的北向 API 调用、Postman 数据提取以及 Python 脚本应用是核心卖点，比传统只会敲命令行的网工更具代际优势。
  2. 高级现场交付专家 / 技术客户经理 (TAM)：面向头部企业客户，您拥有 12,000+ 问题单的实战底气和拉通几十人研发团队的强沟通力，极适合大厂的重大项目技术总舵手。
  3. SRE（站点可靠性工程师 - 网络/基础设施方向）：您精通服务器硬件（RAID/Bond）、Linux 底层调优、数据库表操作和大型网络架构，完全具备转型互联网大厂基础设施 SRE 的核心底座。
